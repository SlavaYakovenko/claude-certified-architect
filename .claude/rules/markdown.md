## Lectures Markdown standards
- All files must be on english langueage only
- All code examples should contain strings and comments on english langueage only
- Don't use emoji (✅, ❌, etc.) and symbols that cannot be typed on a standard keyboard (▼, ↓, →, ←, — long dash). Use ASCII equivalents instead: ->, <-, v, -, [OK], [x], etc.
- Box-drawing characters (│, ─, ┌, ┐, └, ┘, ├, ┤, etc.) are allowed for ASCII diagrams.
